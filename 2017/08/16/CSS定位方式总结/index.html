CSS 定位方式总结

Absolute, Relative, Fixed Positioning: How Do They Differ?

Learn CSS Positioning in Ten Steps

CSS 中有四种定位方式， 分别是: static, relative, absolute 和 fixed。要理解这四种定位方式，首先要明确一个概念，就是所有的元素都可以看成是一个方块(CSS box model)。

static

static是默认定位方式，不同的元素有不同的默认值。是最基本的定位方式，即元素按照它该有的样子出现在页面的某个位置。

relative

relative是最容易混淆的。它真正的意思是“relative to itself”。如果只是对一个元素设置position: relative;而没有设置具体的偏移量(top, left, bottom, right)， 那么元素的定位仍然相当于 static。如果给元素一个偏移量，比方说 top: 10px,那么元素就会在 static 的常规位置基础上向下便宜10px。关于relative还有两个很重要的用法：

  1. 可以在 relative 的基础上设置 z-index，来定义元素显示的层序(谁覆盖在谁上面)
  2. relative配合absolute使用可以限制absolute的作用范围。 relative元素的子元素如果定位方式为absolute，那么子元素可以在relative元素中任意定位。例子。这点非常有用！

absolute

这就非常简单啦。如果没有一个relative的父元素，以absolute定位的元素在页面中的位置是固定不变的，即不会影响别的元素，也不会被别的元素影响，会被移除出元素树单独渲染。合理使用absolute会减少页面的reflow。

fixed

这种定位方式是 relative to viewpoint。即元素会固定在浏览器视窗的某个位置，不会随着 scroll down而改变位置。
